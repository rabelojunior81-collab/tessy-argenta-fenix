# Auditoria Holística do Repositório Master

**Data:** 2026-04-22  
**Escopo:** repo master `tessy-argenta-fenix`

## Leitura executiva

Este repositório deve funcionar como um hub agregador do ecossistema Rabelus. A superfície pública precisa comunicar claramente que os repositórios individuais continuam independentes e que este projeto serve como ponto de entrada, curadoria e arquivo visual.

## Prioridades

### Urgente

1. Publicar a landing page principal em `index.html`.
2. Manter o README como guia de navegação do hub.
3. Garantir que os repositórios individuais não sejam sobrescritos.

### Alta prioridade

1. Preservar os ativos visuais do `repo-manus-hackathon` em um arquivo local.
2. Separar claramente arquivo visual, documentação e histórico legado.
3. Adicionar documentos básicos de projeto open source.

### Backlog estratégico

1. Criar uma publicação automatizada para GitHub Pages.
2. Reduzir o peso dos ativos em uma versão futura otimizada para web.
3. Integrar um índice dinâmico dos sub-repositórios e suas fases.

## Riscos observados

- Misturar conteúdo do hub com código de um módulo específico.
- Vazar arquivos locais da máquina do desenvolvedor.
- Perder a clareza entre histórico legado e a nova identidade do master repo.

## Conclusão

O caminho correto é tratar este repositório como uma camada de orquestração e apresentação, não como substituto dos módulos individuais.
