## PREMISSA

Construir um produto de qualidade sem time é possível até certo ponto — especialmente com IA. Mas escalar, vender, suportar e evoluir simultaneamente requer pessoas. O desafio: não há budget para salários. A solução: atrair pessoas que acreditam no projeto o suficiente para trabalhar por participação futura.

O Hackathon é o mecanismo de captura. O modelo de equity é o mecanismo de retenção.

**Princípio central:** O melhor filtro de qualidade de colaborador é dar a ele um problema real para resolver. Hackathon faz isso publicamente, com custo zero.

## PARTE 1 — ESTRATÉGIA DE HACKATHON

### 1.1 Objetivo do Hackathon Tessy

O hackathon não é sobre gerar features rápido. É sobre:

1. **Identificar talentos** dispostos a trabalhar com o ecossistema Tessy
2. **Criar comunidade** ao redor do produto
3. **Gerar conteúdo** orgânico (participantes postam sobre o que constroem)
4. **Validar posicionamento** — o que as pessoas constroem com Tessy diz muito sobre o que ela é para eles
5. **Recrutar co-workers** para o projeto de longo prazo

### 1.2 Formato Recomendado — "Tessy Build Week"

**Nome:** Tessy Build Week
**Duração:** 7 dias (online, assíncrono)
**Periodicidade:** Trimestral (a cada 3 meses)
**Formato:** Assíncrono — cada participante constrói no seu ritmo, sem horário fixo

**Por que assíncrono:**
- Alcança pessoas com emprego fixo (nosso público principal)
- Menos barreira de entrada que eventos ao vivo
- Participantes no Brasil inteiro, sem limitação geográfica
- Conteúdo gerado (posts, vídeos) acontece ao longo de 7 dias = marketing distribuído

### 1.3 Trilhas de Desafio

**Trilha 1 — Builder** (para devs)
> "Use Tessy para construir algo que você usaria todo dia. Pode ser um workflow, uma integração, uma automação."

**Trilha 2 — Integrador** (para devs mais avançados)
> "Crie uma integração nova para o Tessy — um novo provider LLM, um novo viewer, uma nova tool de IA."

**Trilha 3 — Documentador** (para quem escreve bem)
> "Crie um tutorial completo, guia de uso ou caso de uso real com Tessy. Pode ser em vídeo ou texto."

**Trilha 4 — Designer** (para UX/UI)
> "Reimagine uma tela do Tessy ou proponha um novo componente de interface para a visão Colmeia."

### 1.4 Premiação (sem dinheiro necessário)

| Posição | Prêmio |
|---------|--------|
| 1º lugar cada trilha | Contribuidor Fundador — equity simbólica + badge permanente + voz ativa no roadmap |
| Top 3 cada trilha | Acesso vitalício ao Tessy Pro quando lançar + menção pública |
| Todos os participantes | Tessy Early Builder Badge + acesso antecipado a features |
| Destaque especial | Conteúdo do participante promovido nos canais da Tessy |

**Nota sobre equity simbólica:** Não precisa ser equity real no início. Pode ser um "Founders Circle" com benefícios reais: acesso a todas as features, voz no roadmap, créditos LLM, e um mecanismo formal de conversão para equity real se o projeto atingir receita X.

### 1.5 Infraestrutura do Hackathon (custo zero)

| Componente | Ferramenta | Custo |
|-----------|-----------|-------|
| Inscrições | Google Forms ou Tally.so | Gratuito |
| Comunicação | Discord (servidor Tessy) | Gratuito |
| Submissões | GitHub (PR no repo de showcases) | Gratuito |
| Votação | Discord reactions ou formulário | Gratuito |
| Transmissão de resultados | Twitter/X Spaces ou YouTube Live | Gratuito |

### 1.6 Divulgação do Hackathon

**3 semanas antes:**
- Anúncio no Twitter/X com visual impactante
- Post no LinkedIn com call to action para devs
- Divulgação nos Discords: Rocketseat, Dev Samurai, Código Livre BR, Indie Hackers BR

**1 semana antes:**
- Reminder com stories de quem já se inscreveu
- Post com "o que você pode ganhar" (benefícios concretos)

**Durante:**
- Daily update no Discord com projetos em andamento
- Encorajar participantes a postarem no Twitter com hashtag #TessyBuild

**Pós-hackathon:**
- Post de resultados com showcase dos projetos
- Entrevista curta com os vencedores (vídeo de 5min no YouTube)
- Convite direto para os melhores colocados sobre colaboração de longo prazo

## PARTE 2 — BOOTSTRAP DE TIME

### 2.1 Os Perfis Que Você Precisa (em ordem de prioridade)

| Perfil | Por quê | Onde encontrar |
|--------|---------|---------------|
| **Dev Full-stack** (1-2 pessoas) | Para executar missões técnicas enquanto você tem visão estratégica | Hackathon, GitHub, Discord |
| **Dev de Conteúdo / DevRel** (1 pessoa) | Para criar tutoriais, posts, vídeos — conteúdo técnico que converte | Twitter/X dev BR, YouTube dev |
| **Designer UX/UI** (1 pessoa) | Para elevar o LiquidGlass e criar materiais de marketing | Behance BR, Dribbble, hackathon |
| **Operações / Growth** (1 pessoa) | Para cuidar de comunidade, affiliate tracking, parcerias | LinkedIn, indicação |

### 2.2 Como Identificar as Pessoas Certas

**Sinal 1 — Eles aparecem sem ser chamados:**
A pessoa que comentou no seu post, fez uma pergunta inteligente no Discord, ou abriu uma issue no GitHub sem ser pedida é candidata natural.

**Sinal 2 — Eles fazem, não só falam:**
No hackathon, quem entrega — independente da qualidade — demonstra disposição para execução. Quem só se inscreveu mas não entregou não é o perfil bootstrap.

**Sinal 3 — Eles entendem a visão:**
Quando você explica o que é Tessy e eles completam a frase com algo que você não tinha pensado, é sinal de alinhamento.

**Sinal 4 — Eles têm skin in the game:**
Perguntam sobre o negócio, sobre monetização, sobre como crescer — não só sobre a feature X. Esses são os que potencialmente viram sócios.

### 2.3 A Conversa Inicial (sem prometer o que não pode cumprir)

Script para a conversa com um potencial colaborador bootstrap:

> "Eu estou construindo a Tessy — vou te mandar o link, mas resumindo: é um ambiente de dev com IA, terminal real e LLM gateway, focado no mercado brasileiro. Estou no estágio de validação, sem receita ainda mas com produto funcional e uma visão clara. Não posso pagar salário agora. Mas se você acreditar no que estou construindo, eu gostaria de conversar sobre trabalhar juntos com participação futura — que começa simbólica e formaliza conforme o projeto crescer. Sem compromisso de nada agora, só uma conversa."

O que não fazer:
- Não prometer equity específica antes de formalizar
- Não pedir dedicação full-time antes de construir confiança
- Não criar expectativas de receita sem base real

### 2.4 Estrutura de Contribuição Progressiva

**Fase 0 — Contribuidor Voluntário** (sem compromisso)
- Contribui quando pode, sem prazo
- Recebe: acesso antecipado, badge de contribuidor, feedback direto do fundador
- Não assina nada
- Duração: 4-8 semanas de observação mútua

**Fase 1 — Colaborador de Projeto** (compromisso leve)
- Entrega X deliverables por mês (definido juntos)
- Recebe: Tessy Pro vitalício + participação em pool de equity simbólica
- Assina: Contributor Agreement simples (template no GitHub)
- Duração: 3-6 meses

**Fase 2 — Co-fundador / Equity Partner** (compromisso real)
- Dedicação parcial ou total ao projeto
- Recebe: equity formal + eventual salário quando MRR permitir
- Assina: Shareholder Agreement ou Vesting Agreement formal
- Critério de entrada: produto com MRR > R$5.000 ou investimento externo

## PARTE 3 — MODELO DE EQUITY

### 3.1 Princípios do Modelo

1. **Equity não é salário** — não é pagamento por trabalho. É aposta no futuro do projeto.
2. **Vesting protege todos** — tanto o fundador quanto o colaborador
3. **Comece simbólico, formalize com tração** — não distribuir equity de empresa inexistente antes de ter estrutura legal
4. **Transparência total** — todo colaborador sabe o cap table desde o início

### 3.2 Estrutura Sugerida (pré-investimento)

| Stakeholder | % sugerida | Condição |
|-------------|-----------|----------|
| Adilson (Fundador) | 70-75% | Founder shares |
| Co-fundador técnico (se houver) | 10-15% | 4 anos vesting, cliff 1 ano |
| Pool de colaboradores (hackathon, devrel, etc.) | 5-10% | Distribuído conforme contribuição |
| Advisor pool | 2-5% | Para mentores e advisors estratégicos |
| Futuro investidor/acelerador | 10-20% | Reservado (não emitido ainda) |

### 3.3 Vesting Schedule Padrão

Para qualquer co-fundador ou colaborador sênior:

```
Total: X%
Cliff: 12 meses (se sair antes, perde tudo)
Vesting: 4 anos mensal após o cliff
Aceleração: Em caso de aquisição ou rodada de investimento,
            vesting total ou parcial é acelerado
```

### 3.4 Formalização Legal (por fases)

**Agora (sem custo):**
- Contributor Agreement — documento que especifica: contribuição esperada, benefícios oferecidos, e intenção de formalizar equity no futuro. Não é equity real, mas cria vínculo e compromisso.
- Template disponível: GitHub/open-source contributor agreements adaptados

**Quando MRR > R$3.000:**
- Abrir MEI ou ME como entidade legal
- Formalizar relação com co-fundadores via acordo de sócios
- Custo: R$500-2.000 em honorários advocatícios

**Quando MRR > R$10.000 ou investimento externo:**
- Constituição formal de empresa (LTDA ou SA)
- Cap table formal com vesting agreements assinados
- Advogado especializado em startups

### 3.5 Advisor Equity — Alavancagem Estratégica

Além de co-workers, advisors podem abrir portas sem precisar de salário. Modelo:

- **Advisor operacional** (1-2% / 2 anos vesting): alguém que faz introductions, revisa estratégia mensalmente, conecta com investidores
- **Advisor técnico** (0.5-1%): referência técnica que valida decisões de arquitetura
- **Advisor de mercado** (0.5-1%): alguém com acesso ao mercado alvo (líderes de comunidades dev BR, ex-CPOs de startups)

**Perfis de advisor a buscar:**
- Ex-fundadores de startups de ferramentas para devs no Brasil
- Líderes de comunidades (Rocketseat, Dev Samurai, etc.)
- Investidores-anjo que já investiram em ferramentas dev/IA
