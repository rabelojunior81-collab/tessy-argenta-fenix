## CONTEXTO DE MERCADO

### O Fenômeno "Vibe-trabalhar" (2025-2026)

O mercado passou por uma transição sem precedentes: pessoas deixaram de **comprar SaaS** e passaram a **construir suas próprias ferramentas**. Os drivers:

- **Claude Code, Cursor, Lovable, Bolt.new, Opencode** — democratizaram a construção de software
- **OpenRouter, Gemini free tier, Mistral free** — zeraram o custo de barreira para LLMs
- **Vibe Coding** — virou mainstream: você descreve, a IA constrói

Isso criou um público totalmente novo: o **Builder-Consumer** — não é dev sênior, não é usuário de SaaS genérico. É alguém que constrói ferramentas para si mesmo (e para vender).

### O Mapa Competitivo Real


## SEGMENTOS DE PÚBLICO

### Segmento A — Semi-pro Developer Brasileiro (PRIORITÁRIO)

**Quem é:**
- Dev com 1-4 anos de experiência
- Trabalha ou freelança com desenvolvimento
- Usa ferramentas como VS Code, GitHub, npm
- Já tentou Claude Code ou Cursor mas achou "pesado" ou "caro"
- Paga em reais, sente o dólar
- Ativo em Discord, YouTube, Twitter/X brasileiro

**Dores específicas:**
- Ferramentas poderosas custam USD 20-40/mês (R$100-200) — caro para BR
- CLI puro intimida — quer interface visual
- Não quer mandar código para cloud (privacidade)
- Tem GitHub, precisa de integração real
- Quer terminal real, não sandbox

**Como Tessy resolve:**
- Browser-native, sem instalação complexa
- Terminal PTY real, filesystem local
- GitHub nativo (read/write/PR)
- Vault criptográfico — zero cloud para seus dados
- Futuramente: preço em BRL, acessível

**Mensagem central para este segmento:**
> "Tessy é o Claude Code com interface — controle real, sem aprender CLI, sem mandar código pra nuvem."

**Onde encontrá-lo:**
- Discord: Código Livre BR, Rocketseat, Dev Samurai
- YouTube: comentários de vídeos sobre Claude Code, Cursor, IA para devs
- Twitter/X: #DevBR, #VibeCode, comunidades de indie makers BR
- Reddit: r/brdev, r/programming_br
- LinkedIn: grupos de developers brasileiros

**Jornada de aquisição:**
```
Descoberta (conteúdo) → Curiosidade (demo vídeo) → Uso gratuito →
→ Valor percebido (terminal + GitHub real) → Conversão paga
```

### Segmento B — Indie Maker / SoloPreneur Técnico (SECUNDÁRIO, ALTO VALOR)

**Quem é:**
- Constrói seus próprios produtos de software
- Vende micro-SaaS, plugins, ferramentas
- Tem múltiplos projetos simultâneos
- Paga por ferramentas que economizam tempo real
- Está no ecossistema de Indie Hackers, Product Hunt
- Tem renda, pode pagar mais

**Dores específicas:**
- 10+ API keys espalhadas sem organização
- Não sabe quanto gasta por mês em LLMs
- Precisa mudar de modelo LLM conforme o projeto
- Quer workflow de dev integrado (chat + código + git + terminal)
- AutoDoc — quer que a IA saiba o que as libs que usa fazem

**Como Tessy resolve:**
- Vault centralizado para todas as API keys
- (Futuro) Cost tracking por projeto
- (Futuro) Multi-provider LLM
- Workspace com FS + git + terminal + IA em uma tela
- AutoDoc scheduler

**Mensagem central:**
> "Tessy é o único ambiente que junta suas API keys, seu filesystem, seu git e seu LLM em um único lugar — com criptografia real."

**Onde encontrá-lo:**
- Indie Hackers (global, mas comunidade BR ativa)
- Product Hunt — lançamento estratégico
- Twitter/X: #indiehacker, #makersoftwitter, comunidade BR
- Newsletter: Indie Makers BR, SaaS Hackers
- Discord: Indie Hackers Brasil

**Jornada de aquisição:**
```
Product Hunt launch → Trail viral (feature única) →
→ Uso intenso (múltiplos projetos) → Plano pago (ROI claro)
```

### Segmento C — Dev Sênior / Tech Lead (NICHO PREMIUM, LONGO PRAZO)

**Quem é:**
- Usa Claude Code e/ou Cursor profissionalmente
- Tem time e precisa de protocolo de trabalho com IA
- Valoriza metodologia (TDP/TSP/TMP têm apelo real)
- Pode pagar plano enterprise
- Evangeliza ferramentas para o time

**Dores específicas:**
- IA sem metodologia = código sem padrão
- Não tem como passar contexto do projeto para o LLM de forma consistente
- Time usa LLMs diferentes sem coordenação
- Queria um "Claude Code com memória institucional"

**Como Tessy resolve:**
- TDP/TSP/TMP como sistema de trabalho com IA
- AutoDoc mantém contexto atualizado automaticamente
- Workspace com git integrado = contexto de projeto real
- (Futuro) Multi-usuário / compartilhamento de workspace

**Mensagem central:**
> "Tessy é o único ambiente com metodologia de IA nativa — seus projetos têm memória, protocolo e rastreabilidade."

**Onde encontrá-lo:**
- LinkedIn: Tech Leads, Engineering Managers brasileiros
- Eventos: The Developer's Conference (TDC), QCon SP, BrazilJS
- Podcasts: Hipsters.tech, Código Fonte TV, DevPodcast

## ESTRATÉGIA DE LANÇAMENTO (Fases)

### Fase 0 — Stealth + Validação (Semanas 1-4)

**Objetivo:** Encontrar os primeiros 20 usuários "verdadeiros" — pessoas que usam Tessy semanalmente sem serem pedidas.

**Ações:**
- Abrir Early Adopter Program (ver Documento 04)
- 5 posts de conteúdo de alta qualidade no Twitter/X e LinkedIn
- DM manual para 30-50 perfis alinhados com Segmento A
- Nenhuma propaganda — só conversas reais sobre dores

**Métrica de sucesso:** 20 usuários ativos por semana

---

### Fase 1 — Tração Orgânica (Meses 1-3)

**Objetivo:** 100 usuários ativos, 3 evangelistas que falam de Tessy sem serem pedidos.

**Ações:**
- 2 posts/semana em formato "como eu resolvi X com Tessy" (ver Documento 05)
- 1 vídeo demo no YouTube por mês
- Participação ativa em 3-5 comunidades Discord BR
- Lançamento do programa de afiliados LLM (ver Documento 02)
- 1 hackathon online para construir features com a comunidade (ver Documento 03)

**Métrica de sucesso:** 100 usuários, 10 menções orgânicas, primeiros R$500 em afiliados

---

### Fase 2 — Validação Comercial (Meses 3-6)

**Objetivo:** Provar que pessoas pagam pelo produto.

**Ações:**
- Lançar "Tessy Indie" — plano pago em BRL (ver Documento 07)
- Product Hunt launch — coordenado com comunidade de early adopters
- Parceria com 1-2 plataformas de ensino BR (ver Documento 06)
- Abrir multi-provider LLM como feature diferenciadora

**Métrica de sucesso:** 50 pagantes, MRR de R$2.000+

---

### Fase 3 — Escala (Meses 6-12)

**Objetivo:** 500+ usuários, time formado, produto reconhecido no mercado BR.

**Ações:**
- Presença em eventos físicos (TDC, BrazilJS) com demo ao vivo
- Hackathon presencial em SP ou RJ
- Programa de equidade para contribuidores ativos (ver Documento 03)
- Expansão para LATAM (Argentina, México, Colômbia)

**Métrica de sucesso:** 500 usuários, 200 pagantes, time de 3-5 pessoas

### O "Momento Aha" que Tessy precisa entregar

O usuário precisa ter um "momento aha" nos primeiros 10 minutos:

1. Abre Tessy no browser
2. Conecta o GitHub
3. Pede para a IA: "liste os arquivos do meu repo e me explique a arquitetura"
4. IA usa as tools nativas, navega o repo, responde em contexto
5. Usuário abre o terminal real, roda `npm install`, vê o output
6. **Aqui é o momento aha**: "isso é diferente de tudo que usei antes"

**Toda a estratégia de conteúdo deve ser construída ao redor deste momento.**
