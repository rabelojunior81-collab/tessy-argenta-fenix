<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Root commit -> sync engine | Root Git history determines which module repos may be mutated | changed paths, commit message, module target |
| Sync engine -> module repos | Automation stages, commits, and pushes into nested repos | file diffs, sync commit, remote push |
| Tracked hook -> local Git hook | Installed hook delegates local post-commit execution to repo-tracked logic | script path, environment, bypass flags |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-A | Tampering | module replication | mitigate | Restrict staged paths to the selected module repo and verify repo state before acting. |
| T-04-02-B | Infinite loop / duplicate execution | post-commit hook | mitigate | Keep hook delegation thin and guard against recursive re-entry in the sync engine. |
| T-04-02-C | Unsafe divergence handling | outbound sync | mitigate | Detect dirty/ahead/behind states and stop instead of merging automatically. |
</threat_model>
