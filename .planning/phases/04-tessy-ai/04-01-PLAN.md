<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Repo docs -> operator behavior | Documentation shapes how future sync commands are used | topology rules, conflict policy, command flow |
| Tracked config -> install script | Config determines which repos the installer prepares | module paths, remotes, hook policy |
| Installer -> local Git metadata | Bootstrap changes root Git settings and hook state | remote URL, hook entry points, support dirs |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-A | Misconfiguration | installer | mitigate | Require explicit reporting of current vs desired root origin before changing it. |
| T-04-01-B | Drift | docs/config | mitigate | Keep module mapping and operator docs tracked in the repo and sourced from the same contract. |
| T-04-01-C | Accidental exclusion | .gitignore | mitigate | Review ignore changes so sync artifacts are ignored without hiding required planning or agent files. |
</threat_model>
