## Implementation Details

### OpenAIStyle
- Wire format: SSE with `data: {...}` JSON events
- Uses `eventsource-parser/stream` (NOT native EventSource API)
- Yields `text` chunks, `tool_call_delta` fragments, `usage`, and `done`
- Default model: `gpt-4o`

### AnthropicStyle
- Wire format: SSE with named events (`event: content_block_delta`)
- Handles `message_start`, `content_block_delta`, `message_stop` events
- `delta.type === 'input_json_delta'` yields tool call fragments with `argDelta`
- Default model: `claude-sonnet-4-6`

### GeminiStyle
- Wire format: SSE with `?alt=sse` query param (MANDATORY)
- IMPORTANT: Gemini sends COMPLETE `functionCall` objects per chunk, NOT deltas
- Yields `tool_call_complete` directly (not `tool_call_delta`)
- Default model: `gemini-2.5-flash`

### OllamaStyle
- Wire format: NDJSON (newline-delimited JSON, NOT SSE)
- Uses `TextDecoderStream` + readline-style line parsing
- Complete objects per line via `message.tool_calls` array
- Checks `done: true` flag to terminate stream
- Default model: `qwen2.5-coder:14b`

### OpenRouterStyle
- Delegates to OpenAIStyle with custom baseUrl
- Base URL: `https://openrouter.ai/api/v1/chat/completions`
- Adds OpenRouter-specific headers: `HTTP-Referer`, `X-Title`
- Uses eventsource-parser for SSE parsing (same as OpenAI)

### ProviderRegistry
- Static factory with `register()`, `get(id)`, `list()` methods
- Pre-registers all 5 providers
- `get()` throws descriptive error with available providers list
- `list()` returns `Array<{id: string, displayName: string}>` for UI
