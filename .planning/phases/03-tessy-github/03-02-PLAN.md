<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| GitHub service -> viewer | Remote repo metadata shapes visible browsing state | repo tree, branch names, file contents, errors |
| Project override -> global connection | Project-local repo selection changes effective target | global repo, project repo, effective repo |
| File open policy -> viewer | Large file detection determines whether a file opens directly | file metadata and content size |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-A | Tampering | GitHubContext | mitigate | Separate global repo from effective repo and expose override precedence explicitly. |
| T-03-02-B | Information Disclosure | GitHubViewer | mitigate | Keep the viewer native and compact; do not surface raw token or secret metadata. |
| T-03-02-C | DoS / Friction | Tree rendering | mitigate | Preserve loading states and lazy tree expansion so large repos stay usable. |
</threat_model>
