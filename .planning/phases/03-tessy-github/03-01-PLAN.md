<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Browser storage -> React state | Session metadata influences auth and request behavior | auth session, provider choice, expiry metadata |
| GitHub API -> service layer | Remote responses shape repo access and auth state | repo metadata, contents, token errors |
| UI modal -> auth storage | User-entered credentials are captured for later requests | PAT or auth code exchange result |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-A | Information Disclosure | authProviders | mitigate | Keep raw tokens out of UI state and out of copy/tooltip surfaces. |
| T-03-01-B | Spoofing | GitHub auth path | mitigate | Treat OAuth App PKCE as unsupported and do not build a flow that depends on it. |
| T-03-01-C | Tampering | githubService request layer | mitigate | Centralize headers and API versioning so requests stay consistent. |
</threat_model>
