<threat_model>
## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Preference storage -> UI | Persisted mode preference changes action friction | guided/direct mode, last choice |
| Modal -> mutation service | User confirmation triggers write operations | repo, branch, commit message, PR metadata |
| Pending queue -> viewer | Queue state changes visible action badges and approvals | action type, status, target metadata |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-A | Abuse of direct mode | GitHub action flow | mitigate | Keep destructive action confirmation explicit even when YOLO is enabled. |
| T-03-03-B | Confusing state | Action modal | mitigate | Show repo, branch, source, destination, and worktree target in the modal header/body. |
| T-03-03-C | Stale UI after mutation | Viewer refresh | mitigate | Force a targeted refresh after each approved or executed action. |
</threat_model>
